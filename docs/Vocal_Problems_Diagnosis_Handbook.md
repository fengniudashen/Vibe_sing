## 第一部分：声门层问题（声带本体）

### 1.1 挤卡 / 声门过紧

| 项目 | 详情 |
|------|------|
| **声乐术语** | 挤卡（中）、Vocal Squeeze / Constriction（英）、喉堵（俗语） |
| **体感描述** | "喉咙发紧"、"声音像被掐住"、"唱完嗓子痛"、"高音上不去就用力挤" |
| **好发场景** | 换声点附近（男声 E4-G4，女声 B4-D5）；突然放大音量；紧张/怯场时 |
| **发生率** | 约 90% 的未受训歌手在高音区会出现不同程度的挤卡 |

#### 物理成因（详解）

```
正常高音发声：
  甲杓肌（TA）放松 → 环甲肌（CT）拉伸声带 → 声带变薄变长
  → 基频升高 → 只有边缘振动 → 轻松的头声/混声

挤卡发声：
  甲杓肌（TA）不放松 → 同时环甲肌（CT）也在拉伸
  → 两组肌肉"拔河" → 声带既厚又紧
  → 声门闭合压极高 → 假声带（室带）被迫参与内收
  → 喉室塌陷 → 声门上方狭窄
  
  结果：
  ① 声带振动模式不规律 → 噪声成分大增
  ② 声门上气压异常高 → 声音听起来"憋"
  ③ 频谱在 1-3kHz 出现异常隆起（假声带振动叠加）
```

#### 声学签名（完整指标体系）

| 声学指标 | 正常范围 | 挤卡时 | 检测优先级 | 端侧可行性 |
|---------|---------|--------|-----------|-----------|
| **HNR** | > 18 dB | **< 12 dB** | ⭐⭐⭐⭐⭐ | ✅ 简单高效 |
| **CPP** | > 8 dB | **< 5 dB** | ⭐⭐⭐⭐ | ⚠️ 端侧不稳定 |
| **Jitter** | < 1% | **> 2%** | ⭐⭐⭐ | ✅ |
| **Shimmer** | < 3% | **> 5%** | ⭐⭐⭐ | ✅ |
| **H1-H2** | -2 ~ -6 dB | **< -8 dB** | ⭐⭐⭐ | ✅ 需 F0 |
| **频谱 1-3kHz 鼓包** | 平滑 | **异常隆起** | ⭐⭐⭐ | ✅ CNN 隐式学习 |
| **声门接触商 CQ** | 0.4-0.6 | **> 0.7** | ⭐⭐⭐⭐ | ❌ 需要 EGG |

#### Vibesing 诊断技术路线

```
┌─────────────────────────────────────────────────────────┐
│ V0.1 (Rule-based) — 当前实现                              │
│                                                          │
│   Input: HNR(dB), Energy(dB), F0(Hz)                    │
│   Logic:                                                 │
│     if Energy < -40dB → skip (silence)                  │
│     if F0 < 50Hz → skip (unvoiced)                      │
│     squeeze_prob = sigmoid(0.5 × (12 - HNR))            │
│     if F0 > 220Hz → squeeze_prob × (1.0 ~ 1.3)         │
│     EMA smooth (α=0.4)                                   │
│     State filter: 6 frames ON, 10 frames OFF             │
│   Output: binary (红灯/绿灯)                              │
│   准确率预期: 65-75%                                      │
└─────────────────────────────────────────────────────────┘
                        ↓ 升级
┌─────────────────────────────────────────────────────────┐
│ V0.2 (DL Model)                                         │
│                                                          │
│   Input: Mel[80×32] + HNR + Energy = [82×32]            │
│   Model: 4-layer Causal DWSep CNN (300K params)          │
│   Output: squeeze_prob (sigmoid)                         │
│   新增能力: 从 Mel 频谱中学习 1-3kHz 鼓包等微妙模式        │
│   准确率预期: 80-90%                                      │
└─────────────────────────────────────────────────────────┘
                        ↓ 升级
┌─────────────────────────────────────────────────────────┐
│ V1.0 (Multi-task DL)                                    │
│                                                          │
│   Input: Mel[80×32] + [CPP, HNR, H1-H2, Energy, Slope] │
│   Model: Dual-stream (Mel-CNN + Physics-MLP) Late Fusion │
│   Output: squeeze_prob + 区分"挤卡 vs 漏气 vs 大白嗓"     │
│   新增能力: 与其他病灶的联合判别                            │
│   准确率预期: 90-95%                                      │
└─────────────────────────────────────────────────────────┘
```

#### 产品表现

| 版本 | 呈现方式 |
|------|---------|
| V0.1 | 🔴 红灯（全屏警告）+ 脉冲振动 + 分数骤降 |
| V1.0 | 🔴 红灯 + 文字"喉部紧张——试着放松喉咙" + 时间线标注 |
| V2.0 | 🔴 红灯 + 语音提示"放松...对了，就是这样" + 练习推荐 |

#### 训练矫正建议（App 推荐给用户的练习）

1. **唇颤音（Lip Trill）**：从低到高滑音，强迫放松外喉肌
2. **哼鸣（Humming）**：轻声哼，感受面罩振动而非喉部紧张
3. **"Nay" 混声练习**：用 "Nay"（/neɪ/）音从中音区爬升到高音区，咽壁自然窄化
4. **叹气式发声**：从高音往下"叹气"，让声带自然放松

### 1.2 闭合不严 / 漏气

| 项目 | 详情 |
|------|------|
| **声乐术语** | 漏气、闭合不严、气声过重 |
| **体感描述** | "气不够用"、"声音发虚"、"唱不响"、"总觉得有气漏出来" |
| **好发场景** | 高音轻声时；长音尾巴；疲劳后；女声低音区 |

#### 物理成因

```
声带不完全闭合 →
  闭合相（closed phase）缩短 →
  大量气流从缝隙泄出 →
  声波中噪声成分大增 →
  音量下降，声音"虚"
```

#### 声学签名

| 指标 | 正常 | 漏气 | 与挤卡的区别 |
|------|------|------|------------|
| HNR | > 18 dB | **< 10 dB** | 都低，但原因不同 |
| CPP | > 8 dB | **< 4 dB** | 都低 |
| **H1-H2** | -2 ~ -6 dB | **> 0 dB（偏正）** | **关键区分**：挤卡时 H1-H2 很负 |
| CQ (声门接触商) | 0.4-0.6 | **< 0.3** | 挤卡时 > 0.7 |
| 能量 | 正常 | **偏低** | 挤卡时能量偏高 |

#### Vibesing 诊断技术路线

```
V0.1: 不做（与挤卡混淆，HNR 都低，暂时无法区分）
V0.3: 引入 H1-H2 作为区分特征
       if HNR < 12 AND H1-H2 > 0 → 漏气
       if HNR < 12 AND H1-H2 < -6 → 挤卡
V1.0: CNN 从 Mel 中隐式学习区分
```

#### 产品表现

| 版本 | 呈现方式 |
|------|---------|
| V0.3+ | 🟡 黄灯 + "声带闭合不完全——试着增加一点支撑" |

#### 训练矫正建议

1. **"Gee" 闭合练习**：/ɡiː/ 音，利用 /ɡ/ 辅音带动声门闭合
2. **声门起音练习（轻型）**：短促的 "Ah-Ah-Ah"，训练声带快速闭合
3. **Staccato 练习**：短促断音，强化闭合能力

### 1.3 声门冲击 / 硬起音

| 项目 | 详情 |
|------|------|
| **声乐术语** | 硬起音、声门冲击（Glottal Attack / Hard Onset） |
| **体感描述** | 每句开头有"咳"的感觉，声音起始很硬 |
| **好发场景** | 元音开头的乐句；紧张时；急促发声时 |

#### 声学签名

| 指标 | 检测方式 |
|------|---------|
| 起音瞬态能量 | 前 20ms 能量上升斜率 > 阈值 |
| 起音前的声门脉冲 | 频谱中出现极短（<5ms）的宽带噪声峰 |

#### Vibesing 诊断技术路线

```
V1.0+: 事件检测（onset detection）
  检测每句起音时，前 50ms 的能量上升率
  if rise_rate > threshold → "硬起音"
  技术：规则引擎（不需要 DL）
```

#### 产品表现

V1.0+: ⚠️ 文字提示 "起音偏硬——试着用气息带出声音（soft onset）"

### 1.4 假声带参与 / 室带振动

| 项目 | 详情 |
|------|------|
| **声乐术语** | False Fold Phonation / Ventricular Fold Vibration |
| **体感描述** | 声音"粗糙"、"嘶哑"、"有金属质感但不舒服" |
| **好发场景** | 严重挤卡时；吼唱（Growl）；嗓音疲劳后 |

#### 声学签名

频谱在 1-3kHz 出现异常的**次谐波**（subharmonic）或**非整数倍频成分**——因为假声带的振动频率与真声带不同。

#### Vibesing 诊断技术路线

```
V1.0+: CNN 检测频谱中的次谐波模式
  此特征对 Mel 频谱的 CNN 更容易学习（独特的频谱 pattern）
  可以作为"严重挤卡"的子标签
```

## 第二部分：声道层问题（共鸣与位置）

### 2.1 喉位高 / 声道缩短

| 项目 | 详情 |
|------|------|
| **声乐术语** | 喉位高、High Larynx |
| **体感描述** | "声音发尖"、"唱高音时感觉喉咙上提"、"音色很'白'" |
| **好发场景** | 音高上升时（喉头跟着音高走） |

#### 声学签名

| 指标 | 值 | 检测方式 |
|------|---|---------|
| F1/F2/F3 | **全部上移 10-20%** | 通过 LPC 或 CNN 隐式提取 |
| 共振峰偏移一致性 | **> 0.7**（一致上移） | 计算各共振峰相对偏移的变异系数 |
| HNR/CPP | 可正常 | — |

**与挤卡的核心区别**：喉位高时声门层可以是正常的（HNR 正常），问题出在声道层。

#### Vibesing 诊断技术路线

```
V1.0: 
  CNN 从 Mel 频谱中隐式学习共振峰位置
  辅助信号：在训练数据中用 Praat 提取精确 F1/F2/F3 做 auxiliary loss
  Output: high_larynx (sigmoid)
  
V2.0:
  联合 F0 做动态检测：
  if dF1/dF0 > 预期（F1 随音高上移速度过快）→ 喉位在跟着跑
```

#### 产品表现

V1.0+: 🟣 紫灯 + "喉头偏高——试着打哈欠的感觉，让喉咙放松下沉"

---

### 2.2 喉位过低 / 压喉

| 项目 | 详情 |
|------|------|
| **声乐术语** | 压喉、Low Larynx (excessive) |
| **体感描述** | "声音发闷"、"像在嗓子里面唱"、"有假的'浑厚感'" |
| **好发场景** | 过度强调"放下喉头"的学生；模仿低音歌手时 |

#### 声学签名

与喉位高完全相反：F1/F2/F3 **全部下移**，且打哈欠时的极端下移。

#### Vibesing 诊断技术路线

```
V1.0+:
  与喉位高使用相同的共振峰检测逻辑
  if FormantShiftConsistency > 0.7 AND shift direction = DOWN → 压喉
```

### 2.3 鼻音过重 / 软腭下沉

| 项目 | 详情 |
|------|------|
| **声乐术语** | Hypernasality / 鼻音过重 |
| **体感描述** | "闷闷的"、"像感冒了"、"声音出不来" |
| **好发场景** | /m/, /n/, /ŋ/ 鼻辅音后的元音；疲劳时软腭控制力下降 |

#### 声学签名

| 指标 | 正常 | 鼻音过重 |
|------|------|---------|
| **反共振峰 (anti-formant)** | 不明显 | **在 ~800-1200Hz 出现明显凹陷** |
| F1 带宽 | 窄 | **带宽增大**（鼻腔耦合增加了辐射阻抗） |
| A1-P0 (第一共振峰与鼻腔峰的差) | 正常 | **减小** |

#### Vibesing 诊断技术路线

```
V1.0+:
  检测 Mel 频谱中 800-1200Hz 区域的能量凹陷
  可以作为 CNN 的隐式特征学习目标
  困难点：需要知道用户发的是什么元音（不同元音的"正常"频谱不同）
  
  实用简化方案：
  只在非鼻辅音的纯元音段检测
  if 800-1200Hz_energy < 相邻频段能量 × 0.6 → 疑似鼻音过重
```

### 2.4 鼻音不足 / 软腭过高

| 项目 | 详情 |
|------|------|
| **声乐术语** | Hyponasality / Denasality |
| **体感描述** | "声音太亮了但没有温度"、"/m/, /n/ 发不清楚" |

与鼻音过重相反，声学签名是 /m/, /n/ 辅音中鼻腔共振能量不足。

#### Vibesing 诊断技术路线

```
V2.0+: 需要音素级检测，依赖轻量 ASR
优先级低，大部分歌手的问题是鼻音过重而非不足
```

### 2.5 舌根后压 / 咽腔后壁拥堵

| 项目 | 详情 |
|------|------|
| **声乐术语** | Tongue Root Retraction / 舌根紧张 |
| **体感描述** | "声音发闷"、"喉咙深处有东西堵着"、"咽壁不清晰" |
| **好发场景** | 唱 /a/, /o/ 元音时；想"打开喉咙"但方法错误时 |

#### 声学签名

| 指标 | 正常 | 舌根后压 |
|------|------|---------|
| **F2** | 元音标准值 | **下降 200-400 Hz** |
| 语音清晰度 | — | 元音模糊化 |

#### Vibesing 诊断技术路线

```
V2.0: 需要元音识别 + F2 对比
  Step 1: 轻量 ASR 识别当前元音（/a/, /e/, /i/, /o/, /u/）
  Step 2: 查表获取该元音的"正常 F2 范围"
  Step 3: 比较实际 F2 是否显著偏低
  if actual_F2 < expected_F2 - 250Hz → 舌根后压
```

### 2.6 口腔扁塌 / 无纵深共鸣

| 项目 | 详情 |
|------|------|
| **声乐术语** | Flat Oral Cavity / 口腔空间不够 |
| **体感描述** | "声音扁"、"没有立体感"、"像在说话不像在唱歌" |

#### 声学签名

F1/F2 分布平淡，频谱均匀无聚焦（没有突出的共振峰能量集中）。

#### Vibesing 诊断技术路线

```
V1.0+:
  频谱能量分布的"尖峰度"（kurtosis）偏低
  if spectral_kurtosis < threshold → 共鸣缺乏聚焦
```

### 2.7 面罩共鸣缺失

| 项目 | 详情 |
|------|------|
| **声乐术语** | Lack of Mask Resonance / 没有"靠前" |
| **体感描述** | "声音没有穿透力"、"总感觉声音'闷'在里面" |

#### 声学签名

Singer's Formant（~3kHz 附近的能量集中）缺失。F3/F4/F5 能量弱。

#### Vibesing 诊断技术路线

```
V1.0+:
  检测 2.5-3.5kHz 频段相对于全频段的能量占比
  if singer_formant_ratio < threshold → 面罩共鸣不足
  
  注意：训练有素的歌手这个比值会显著高于未受训歌手
  可以作为"声乐水平"的间接指标
```

## 第三部分：气息层问题（动力源）

### 3.1 大白嗓 / 用力过猛

| 项目 | 详情 |
|------|------|
| **声乐术语** | Yelling / Untrained Belting / 大白嗓 |
| **体感描述** | "扯着嗓子喊"、"唱完嗓子疼"、"声音很大但很粗糙" |

#### 声学签名

| 指标 | 正常 Belting | 大白嗓 |
|------|------------|--------|
| H1-H2 | -4 ~ -8 dB | **< -10 dB**（声门闭合过强） |
| Singer's Formant 3kHz | **明显存在** | **完全缺失** |
| 频谱 1-4kHz | 有聚焦峰 | **能量均匀摊平** |
| 音量 | 大 | **很大** |
| HNR | > 15 dB | > 15 dB（可正常，因为谐波结构仍在） |

**关键洞察**：大白嗓和挤卡在 HNR 上可以不同！大白嗓的 HNR 可以正常（因为声门闭合好，谐波纯度高），而挤卡的 HNR 低（因为假声带参与导致噪声）。区分二者的关键是 Singer's Formant。

#### Vibesing 诊断技术路线

```
V1.0:
  检测维度：H1-H2 + Singer's Formant + Energy
  if H1-H2 < -10 AND singer_formant_ratio < threshold AND energy > high
    → 大白嗓
  区分于挤卡：HNR 正常 → 不是挤卡；但 Singer's Formant 缺失 → 不是训练 Belting
```

#### 产品表现

V1.0+: 🟠 橙灯 + "你在用力喊——试着用共鸣代替力量"

---

### 3.2 气息不足 / 支撑塌陷

| 项目 | 详情 |
|------|------|
| **体感描述** | "唱到后面没气了"、"长音维持不住"、"声音越来越虚" |

#### 声学签名

- 帧级能量随时间**单调下降**（在一个乐句内）
- 乐句末尾 HNR 下降（气不够了声带开始闭合不全）
- F0 在乐句末尾微降（气压不足导致音高偏低）

#### Vibesing 诊断技术路线

```
V1.0+:
  检测乐句内的能量衰减斜率
  if energy_slope_per_second < -6dB/s → 气息支撑不足
  需要乐句边界检测（通过能量静音段切分）
```

---

### 3.3 气息不稳 / 横膈膜抖动

| 项目 | 详情 |
|------|------|
| **体感描述** | "声音发抖"、"长音不稳"、"听起来很紧张" |

#### 声学签名

| 指标 | 正常 | 气息不稳 |
|------|------|---------|
| 帧间能量变异系数 (CV) | < 5% | **> 10%** |
| F0 变异（非颤音频率） | < 0.5 semitone | **> 1 semitone** |
| 调制频率 | — | **不规律**（区别于 5-6Hz 的正常颤音） |

#### Vibesing 诊断技术路线

```
V1.0+:
  计算 0.5 秒滑动窗内的 energy_CV 和 f0_CV
  if energy_CV > 0.1 AND f0_modulation 不在 4-7Hz → 气息不稳
```

---

### 3.4 推气 / 声门下压过高

| 项目 | 详情 |
|------|------|
| **声乐术语** | Over-breathing / Pushed Sub-glottal Pressure |
| **体感描述** | "用了很大力气但声音不响"、"气往上冲" |

与大白嗓类似但原因不同：大白嗓是声带主动用力，推气是气息主动用力（声带被动被冲开）。

#### 声学签名

能量极高但 H1-H2 偏正（声带被冲开，闭合不充分）。

## 第四部分：声区转换问题

### 4.1 换声断层 / 破音

| 项目 | 详情 |
|------|------|
| **声乐术语** | Voice Break / Voice Crack / 破音 |
| **体感描述** | "音高突然跳上去了"、"声音断了" |

#### 声学签名

| 指标 | 值 | 检测方式 |
|------|---|---------|
| **F0 帧间差** | **> 50 Hz in 1 frame (16ms)** | 直接计算 |
| 能量 | 可能骤降 | — |
| 频谱 | 突变 | Mel 帧间余弦距离突增 |

#### Vibesing 诊断技术路线

```
V1.0 (规则引擎，不需要 DL):
  if |F0[t] - F0[t-1]| > 50Hz 
     AND both_voiced 
     AND energy > silence_threshold
    → VOICE BREAK EVENT

  产品表现：💥 红色闪烁（瞬态事件，不需要状态机持续）
```

---

### 4.2 换声回避 / 永远用胸声硬撑

| 项目 | 详情 |
|------|------|
| **体感描述** | "就是不肯换声"、"一直喊到破音为止" |

#### 声学签名

在应该进入头声的音域（>E4 男声），H1-H2 仍然保持极负值（胸声特征），且开始出现挤卡信号。

#### Vibesing 诊断技术路线

```
V0.2+:
  联合混声检测 + 挤卡检测：
  if F0 > passaggio AND mix_ratio < 0.3 (pure chest) → "你可以试着混入一些头声"
```

---

### 4.3 换声过早 / 提前进入假声

| 项目 | 详情 |
|------|------|
| **体感描述** | "高音发虚"、"一到中高音就变假声了" |

#### 声学签名

在还不需要换声的音域，H1-H2 突然变正（假声特征），音量骤降。

#### Vibesing 诊断技术路线

```
V0.2+:
  if F0 < passaggio AND mix_ratio > 0.8 (pure head/falsetto)
    → "你不需要这么早换声，试着多保持一些胸声"
```

---

## 第五部分：混声质量问题

### 5.1 混声比例失衡 / 胸声过重

| 项目 | 详情 |
|------|------|
| **体感描述** | "高音听起来很'重'、很'吃力'"、"混声不够轻" |

#### 产品表现

V0.2+: Slider 偏左（橙色区域）+ "混入更多头声——试着减轻声带的接触力度"

### 5.2 混声比例失衡 / 头声过重

| 项目 | 详情 |
|------|------|
| **体感描述** | "高音完了但没有力量感"、"太'虚'了" |

#### 产品表现

V0.2+: Slider 偏右（蓝色区域）+ "可以多一些胸声的支撑——保持核心稳定"

### 5.3 混声不稳定 / 比例波动

#### 声学签名

mix_ratio 的帧间变异系数偏高。

#### Vibesing 诊断技术路线

```
V0.2+:
  计算 mix_ratio 在 0.5 秒窗内的标准差
  if std > 0.15 → "混声不太稳定——试着用气息支撑住"
```

## 第六部分：修饰与控制问题

### 6.1 喉颤音 / 摇音（不自然颤音）

| 正常颤音 | 异常颤音 |
|---------|---------|
| 频率 4.5-6.5 Hz | **< 4 Hz (Wobble)** 或 **> 7 Hz (Tremolo)** |
| 深度 0.5-2 semitones | **> 2 semitones** |
| 规整 | 不规则 |

#### Vibesing 诊断技术路线

```
V1.0+:
  对 F0 轨迹做短时 FFT（~1 秒窗）
  peak_freq = argmax(FFT(F0)) in 2-10 Hz range
  peak_depth = amplitude at peak_freq
  
  if peak_freq < 4.5 → "颤音偏慢（Wobble）"
  if peak_freq > 6.5 → "颤音偏快（Tremolo）"
  if peak_depth > 2 semitones → "颤音幅度过大"
  if regularity < threshold → "颤音不规则"
```

### 6.2 无颤音 / 直声过多

长音尾部完全没有 F0 调制 → 声音听起来"呆板"、"电子"。

```
V1.0+:
  if sustained_note_duration > 1s AND f0_modulation_depth < 0.3 semitone
    → "可以加入一点自然颤音"
```

### 6.3 力度控制差 / 强弱失调

帧间能量跳变频繁且无规律。

```
V1.0+:
  energy_jitter = std(energy[t] - energy[t-1]) over phrase
  if energy_jitter > threshold → "力度控制还需要练习"
```

## 第七部分：肌肉代偿问题

### 7.1 下巴僵硬 / 颞颌关节锁定

| 项目 | 详情 |
|------|------|
| **体感描述** | "张不开嘴"、"唱快歌吐字不清"、"下巴酸" |

#### 声学签名

辅音→元音过渡时间 > 80ms（正常 < 50ms）。

#### Vibesing 诊断技术路线

```
V2.0:
  需要音素级检测 → 依赖轻量 ASR
  检测 C-V (辅音-元音) transition 的持续时间
  if transition_duration > 80ms → "下巴可能偏紧——试着放松"
```

### 7.2 颈部紧张 / 外喉肌过度参与

#### 声学可检测性

**极难从纯音频检测**。外喉肌紧张主要影响喉位，最终通过共振峰偏移间接体现。

```
V1.0: 通过喉位高的检测间接推断
V3.0: 可能需要 EMG 传感器（超出纯音频范畴）
```

### 7.3 耸肩呼吸 / 浅呼吸

#### 声学可检测性

**难以从纯音频检测**。浅呼吸导致肺活量不足，最终通过"乐句长度过短"和"乐句末尾气息塌陷"间接体现。

```
V1.0: 通过"气息不足"检测间接推断
V3.0: 可能需要加速度计（检测肩部动作）
```
