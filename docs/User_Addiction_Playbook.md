## 第一章：上瘾模型框架

### Nir Eyal "Hooked" 模型 × 声乐训练场景

```
    ┌────────────┐
    │   TRIGGER  │ ← "你今天还没练" / "你之前的最高音是 F4"
    └─────┬──────┘
          ↓
    ┌────────────┐
    │   ACTION   │ ← 打开 App，一键开唱（0 思考成本）
    └─────┬──────┘
          ↓
    ┌────────────┐
    │  VARIABLE  │ ← "哇今天绿灯了！" / "分数 78→82！" / "新解锁！"
    │   REWARD   │
    └─────┬──────┘
          ↓
    ┌────────────┐
    │ INVESTMENT │ ← 录音存档 / 练习数据 / 声乐档案 / 社交内容
    └────────────┘
```

### Self-Determination Theory (SDT) 三要素

| 心理需求 | Vibesing 满足方式 | 具体机制 |
|---------|-----------------|---------|
| **胜任感 (Competence)** | 分数上升、绿灯亮起、等级提升 | 量化进步系统 |
| **自主感 (Autonomy)** | 自选歌曲、自选技巧维度、自定义目标 | 个性化练习路径 |
| **归属感 (Relatedness)** | 排行榜、挑战赛、教练评语 | 社交系统 |

### 核心公式

```
上瘾强度 = (进步的可感知性 × 反馈即时性) / (行动成本 + 挫败感)
```

**Vibesing 的终极武器**：实时反馈。没有任何声乐教育产品能做到"唱完一个音就告诉你做对了没有"。这是我们的护城河。

## 第二章：触发系统 — 让用户想起你

### 2.1 外部触发

| 触发类型 | 实现方式 | 时机 | 转化率预期 |
|---------|---------|------|-----------|
| **推送通知** | "你昨天最高音到了 F4，今天试试 F#4？" | 用户通常练习时间 | 8-12% |
| **连续打卡提醒** | "你已经连续练了 6 天，别断了！" | 距上次练习 20h | 15-20% |
| **进步里程碑** | "你的挤卡率降到了 15%！（7 天前 40%）" | 检测到进步时 | 20%+ |
| **好友动态** | "小红练了《夜曲》，挤卡率 8%，比你低！" | 好友完成练习时 | 5-8% |
| **教练消息** | "我听了你昨天的录音，换声点比上周好多了" | 教练批阅后 | 25%+ |

#### 推送文案原则

```
✅ 好的推送：具体 + 数据 + 挑战
   "你的挤卡率连续 3 天下降！今天高音练习只需要 5 分钟"
   
❌ 坏的推送：空洞 + 催促
   "来练习吧！你好久没打开了"
```

#### 推送频率控制

- 每日上限：2 次
- 用户 3 次未响应推送 → 降频到每 3 天 1 次
- 用户关推送 → 转为 App 内 Banner（下次打开时）
- 绝不在 22:00-08:00 推送

### 2.2 内部触发

内部触发是终极目标：用户**自己**想起要练。

| 情绪 | 触发逻辑 | 产品设计 |
|------|---------|---------|
| "我高音不行" | → 挫折 → 想改进 → 打开 Vibesing | 在挫败后推荐最容易获得进步的练习 |
| "这首歌好好听" | → 想唱 → 想唱好 → 打开 Vibesing | App 内曲库 + "试唱评分" |
| "今天要 KTV" | → 想提前练 → 打开 Vibesing | "KTV 模式"：快速热嗓 + 按歌曲练 |
| "好无聊" | → 刷手机 → 看到图标 → 打开 | 主屏 Widget 显示今日进度 |

#### 关键产品设计：主屏 Widget

```
┌──────────────────────────────┐
│  🎤 Vibesing                 │
│  连续打卡：12 天              │
│  今日目标：5min 高音练习       │
│  ████████░░ 80%              │
│  [一键开练]                   │
└──────────────────────────────┘
```

每当用户滑过主屏，就看到自己的进度。不需要打开 App 就产生了触发。

## 第三章：行动系统 — 让用户 0.5 秒内开始练

### 3.1 行动成本分解

Fogg 行为模型：**行为 = 动机 × 能力 × 触发**

用户打开 App 到发出第一个音，每增加一步都会损失 20% 的用户。

### 3.2 极简启动流程

```
用户打开 App
      ↓
[今日推荐练习] ← 基于 AI 分析的个性化内容（不需要用户选择）
      ↓
[一键开练] ← 一个按钮，点了就开始（麦克风权限已在首次授予）
      ↓
[3...2...1...唱!] ← 3 秒倒计时，立刻进入状态
      ↓
实时反馈开始
```

**核心原则**：从打开 App 到开始发声 < 5 秒。

### 3.3 各场景的极简路径

| 场景 | 行动路径 | 操作步数 |
|------|---------|---------|
| **"我就想练 5 分钟"** | 打开 → 一键推荐 → 5min 练习包 | 2 步 |
| **"我想练某首歌"** | 打开 → 搜歌 → 试唱 | 3 步 |
| **"我想看看进步没"** | 打开 → Dashboard（自动显示） | 1 步 |
| **"我想录一段给朋友听"** | 打开 → 录音 → 分享 | 3 步 |

### 3.4 降低心理阻力

| 阻力 | 解决方案 |
|------|---------|
| "我唱得不好，不敢练" | 默认"私密模式"，录音不外传；用分数（0-100）而非"好/差" |
| "要练很久才有效果吧" | "每日 5 分钟"挑战；研究显示每日短时练习比周末长练更有效 |
| "不知道练什么" | AI 自动推荐今日练习（不需要用户做决策） |
| "我在公共场合不方便" | "默念模式"（V2.0）：用气声/哼鸣练习 |

## 第四章：奖励系统 — 让用户练完觉得爽

### 4.1 即时奖励（每秒级）

这是 Vibesing 最大的差异化优势。

| 奖励 | 触发条件 | 感官通道 | 多巴胺强度 |
|------|---------|---------|-----------|
| 🟢 **绿灯亮起** | 挤卡消失 | 视觉 | ⭐⭐⭐⭐⭐ |
| 📈 **分数跳升** | 当前帧优于平均 | 视觉 | ⭐⭐⭐⭐ |
| ✨ **粒子效果** | 连续 5 秒绿灯 | 视觉 | ⭐⭐⭐ |
| 📳 **触感反馈** | 绿灯切换瞬间 | 触觉 | ⭐⭐⭐ |
| 🎵 **和声伴奏** | 声音健康时自动加和声（V2.0） | 听觉 | ⭐⭐⭐⭐⭐ |

#### 绿灯的心理学

```
挤卡时：🔴 红灯 → 用户紧张 → 本能反应是放松 → 声带自然打开
        ↓
放松后：🟢 绿灯 → 用户开心 → "啊原来是这个感觉！" → 肌肉记忆形成
        ↓
反复：   红→绿→红→绿 → 条件反射建立 → 不看 App 也知道对不对
```

这个循环是**世界上最快的声乐反馈回路**。声乐老师需要你唱完一整句才能给反馈，Vibesing 每 16ms 就给一次。

### 4.2 短期奖励（每次练习结束）

| 奖励 | 内容 | 心理效果 |
|------|------|---------|
| **练习总结卡** | 本次分数、最高音、绿灯比例、进步幅度 | 成就感 |
| **对比图** | "7 天前 vs 今天"的挤卡率叠加图 | 进步可视化 |
| **经验值 (XP)** | 每次练习获得 XP，积累升级 | 游戏化 |
| **连续打卡** | 🔥 火焰数字（借鉴 Duolingo/Snapchat） | 不想断 |
| **解锁语录** | "今天比 90% 的上周的你都好" | 正向激励 |

### 4.3 可变奖励 — 上瘾的核心引擎

Nir Eyal 的核心洞察：**可预测的奖励会无聊，不可预测的奖励会上瘾。**

| 可变奖励类型 | 实现方式 | 心理效果 |
|------------|---------|---------|
| **Self 奖励** | 随机解锁成就徽章 / 隐藏彩蛋 | "下一个是什么？" |
| **Tribe 奖励** | 好友排行榜实时变化 / 挑战赛随机匹配 | "我能不能超过他？" |
| **Hunt 奖励** | 每日随机"挑战音高"（未知难度） | "今天是什么？" |

#### 具体设计：每日挑战

```
每天 App 随机生成：
  "今日挑战：连续 10 秒高音绿灯（F4-A4 区间）"

完成后：
  🏆 + 50 XP + 随机奖杯（共 100 种）

没完成：
  "差一点点！你到了 8 秒，明天再来！"
```

#### 具体设计：随机成就徽章

| 成就 | 触发条件 | 稀有度 |
|------|---------|-------|
| "绿灯新手" | 第一次连续 3 秒绿灯 | ⭐ |
| "挤卡终结者" | 连续 3 天挤卡率 < 10% | ⭐⭐ |
| "高音先锋" | 第一次检测到 F5 | ⭐⭐⭐ |
| "铁肺" | 一口气长音 > 15 秒 | ⭐⭐ |
| "凌晨歌手" | 凌晨 2-4 点练习 | ⭐⭐⭐⭐ |
| "百日苦修" | 连续打卡 100 天 | ⭐⭐⭐⭐⭐ |
| "声乐教授" | 所有维度评分 > 90 | ⭐⭐⭐⭐⭐ |

## 第五章：投入系统 — 让用户舍不得走

### 5.1 数据沉没成本

用户在 Vibesing 中积累的数据越多，离开的成本就越高。

| 数据类型 | 积累方式 | 离开代价 |
|---------|---------|---------|
| **声乐档案** | 系统自动建立（嗓音指纹、音域、强弱项） | 去其他 App 要从头检测 |
| **练习历史** | 每次练习记录分数、时长、进步 | 进步曲线消失 |
| **录音库** | 每次练习可保存录音 | 对比素材消失 |
| **成就收藏** | 徽章、等级、连续天数 | 游戏进度消失 |
| **AI 推荐模型** | 越练越了解用户的弱点和习惯 | 个性化推荐从零开始 |

### 5.2 个性化投入

每次用户与 App 互动，产品都变得更适合 TA：

```
练习 10 次后：
  App 知道你的音域是 C3-E4
  App 知道你在 D4-E4 容易挤卡
  App 知道你通常晚上 9 点练
  App 知道你喜欢周杰伦的歌
  
→ 推荐内容完全为你定制
→ 换到另一个 App？从头来。
```

### 5.3 社交投入

| 投入类型 | 实现方式 |
|---------|---------|
| **添加好友** | 互相看到练习记录 |
| **加入练功房** | 小组 PK，有"责任感" |
| **跟教练建立关系** | 教练了解你的历史 |
| **分享内容** | 朋友圈"我在 Vibesing 连续练了 30 天" |

### 6.3 社交裂变机制

| 机制 | 实现 | 效果 |
|------|------|------|
| **好友对战** | A 邀请 B 唱同一首歌，比挤卡率 | 1→2 传播 |
| **练功房** | 3-8 人组队，每日打卡率 > 80% 全队额外 XP | 留存 + 拉新 |

```
K-factor = (每用户邀请数) × (邀请→注册转化率) × (注册→活跃转化率)

目标：K > 1.0（自增长）
实际预期：
  V0.x: K = 0.3-0.5（靠产品力留存，暂不推社交）
  V1.0: K = 0.8-1.2（社交功能上线）
  V2.0: K = 1.5+（社交飞轮转起来）
```

## 第七章：新手 7 天养成路径

### Day 0：下载（首次打开）

```
目标：30 秒内让用户发出第一个音并看到反馈

流程：
  1. [2 秒] Logo 动画
  2. [5 秒] "Vibesing 帮你实时检测唱歌中的紧张"（一句话定位）
  3. [3 秒] 麦克风权限请求（说明理由）
  4. [5 秒] "随便唱两句，我们先听听" ← 零门槛
  5. [10 秒] 用户随便唱
  6. [5 秒] 展示结果："你的嗓音很有潜力！高音区有一点紧张（挤卡），让我们一起改善"
  
  关键设计：
  - 不要问"你想练什么" / "你是什么水平"（增加决策成本）
  - 不要注册/登录（先用后注册）
  - 不要教程（用了才知道）
```

### Day 1：第一次完整练习

```
目标：完成第一个 5 分钟练习并看到分数

系统推送（根据 Day 0 的检测结果定制）：
  "根据昨天的检测，你在 D4-E4 区间容易紧张。今天花 5 分钟，用哼鸣放松一下？"

练习内容：
  1. [1 min] 哼鸣热身（C3-G3，舒适区）
  2. [2 min] 中音区自由唱（自选简单歌曲片段）
  3. [2 min] 高音区挑战（到 D4-E4，实时红绿灯反馈）

练习结束：
  练习总结卡 + "第一天完成！🏆 +100 XP"
  
  🔔 解锁成就："初体验"（⭐）
```

### Day 2：建立习惯锚点

```
系统推送（在 Day 1 练习时间 ±1h）：
  "昨天你在 E4 保持了 3 秒绿灯，今天试试 5 秒？"

设计重点：
  - 目标极小化（行为锚定：3 秒 → 5 秒，微小进步）
  - 给出与昨天的对比
  - 如果目标达成，额外奖励动画
```

### Day 3：引入对比

```
系统推送：
  "你知道吗？你现在的挤卡率已经从 Day 1 的 35% 降到了 25%！"

设计重点：
  - 首次展示"趋势图"（3 个数据点可以画趋势了）
  - 让用户看到：我真的在进步
  - 解锁功能："历史录音对比"
```

### Day 4：引入可变奖励

```
系统推送：
  "今日限时挑战 🎯：连续 8 秒高音绿灯"

设计重点：
  - 第一次出现"每日挑战"（Day 1-3 太早，会分散注意力）
  - 随机奖励：完成后随机获得一个徽章
```

### Day 5：引入社交

```
系统推送：
  "你已经连续练了 5 天！想挑战一下你的朋友吗？"

设计重点：
  - 邀请好友功能首次出现（不在 Day 1 出现，那时用户还不确定产品好不好）
  - 生成"我的 5 天进步卡"，一键分享
```

### Day 6：强化沉没成本

```
系统推送：
  "你的声乐档案已建立 ✅ 音域 C3-E4 | 强项：中低音共鸣 | 弱项：换声点挤卡"

设计重点：
  - 展示 AI 生成的个人声乐档案
  - 让用户感觉"这个 App 真的了解我"
  - 提供："你想修复哪个弱项？"（首次让用户选择练习目标）
```

### Day 7：里程碑 + 钩子

```
系统推送：
  "7 天完成！🏆🔥 你的声音变化"

设计重点：
  - Day 1 vs Day 7 的 A/B 录音播放对比（用户一听就知道进步了）
  - 一周总结卡：
    "总练习时长：42 分钟
     挤卡率改善：-15%
     最高音突破：E4 → F4
     总 XP：1200"
  - 解锁 Pro 功能预告："解锁混声检测？"
  
  🔔 关键设计：
  如果用户练了 7 天，TA 有 90% 的概率继续用下去
  这 7 天是整个用户生命周期中最重要的 7 天
```

## 第九章：流失唤回 — 把走了的人拉回来

### 9.1 流失预警

```
用户行为特征           风险等级       干预措施
─────────────────────────────────────────
连续 2 天未练           🟡 低         推送："你的连续打卡快断了！"
连续 5 天未练           🟠 中         推送："你已经错过了 X XP"
连续 14 天未练          🔴 高         推送："你的声乐档案还在等你回来"
连续 30 天未练          ⚫ 流失       邮件/短信：折扣 + 回归奖励
```

### 9.2 唤回策略

| 策略 | 内容 | 场景 |
|------|------|------|
| **损失厌恶** | "你的连续打卡 23 天纪录快消失了！" | 5-7 天未回 |
| **进步回顾** | "还记得吗？你来的时候挤卡率 40%，走的时候 15%。你有多少进步在等着继续" | 14 天未回 |
| **社交压力** | "你的好友小红今天超过了你的排名" | 7 天未回 |
| **新功能** | "我们新上线了混声检测，来试试？" | 30 天未回 |
| **折扣** | "Pro 半价回归礼" | 30+ 天未回 |
| **回归奖励** | "回归即送 500 XP + 限定徽章" | 14+ 天未回 |

### 9.3 回归体验设计

```
用户离开 30 天后回来：

1. [不要说"好久不见"] → 说"欢迎回来！你的嗓音变化了吗？来唱两句看看"
2. 快速重新评估（30 秒）→ 与离开前对比
3. 如果退步了：温和的鼓励 → "嗓音状态和之前差不多，几天就能恢复"
4. 如果没退步：惊喜 → "哇，你离开这么久但状态还在！"
5. 立刻推荐一个 3 分钟的轻量练习（不要 10 分钟的重型训练，用户刚回来）
```

## 第十章：数据指标体系

### 10.1 北极星指标

```
北极星指标 = 每周至少练习 3 天的用户数（Weekly Active Practitioners, WAP）
```

为什么不是 DAU？因为声乐练习不需要每天打开 App，但需要持续练习才有效果。WAP 最能代表"用户真正在用 Vibesing 改善唱歌"。

### 10.2 核心指标看板

| 层级 | 指标 | 计算方式 | 目标 |
|------|------|---------|------|
| **获取** | 新增注册 | 日级 | 自然增长 + K-factor |
| **激活** | 首日完成一次练习率 | Day 0 练习人数 / 注册数 | > 60% |
| **留存** | D1 / D7 / D30 留存 | — | 80% / 50% / 30% |
| **习惯** | WAP（周活跃练习者） | 每周 ≥3 天练习 | 持续增长 |
| **进步** | 用户加权平均挤卡率改善 | 30日窗口 | 持续下降 |
| **付费** | Pro 转化率 | Pro用户 / 总用户 | 5-8% |
| **传播** | K-factor | 邀请 × 转化 × 激活 | > 1.0 |

### 10.3 报告节奏

| 频率 | 内容 |
|------|------|
| **实时** | 当前在线用户数、实时练习人数 |
| **日报** | DAU、新增、留存、WAP |
| **周报** | 7 日留存趋势、WAP 变化、流失预警 |

## 第十一章：伦理红线

### 11.1 不做的事

| ❌ 不做 | 为什么 |
|--------|--------|
| **不设计"无限刷"机制** | 练歌不是刷短视频，过度练习损伤声带 |
| **不发"你退步了"的推送** | 打击信心 → 流失，永远用正向框架 |
| **不让用户互相听录音（除非本人同意）** | 这是极私密的内容 |
| **不显示全球排名** | 大部分用户会在底部 → 挫败感 → 流失 |
| **不用"你不如别人"作为激励** | 改用"你比昨天的自己好" |

### 11.2 必须做的事

| ✅ 必须做 | 原因 |
|----------|------|
| **练习时长提醒** | 连续练 30 分钟弹出"你今天练得够多了，休息一下嗓子" |
| **声带健康提示** | 检测到异常声质连续 3 天 → "建议去看下嗓音科" |
| **隐私保护** | 录音仅存本地，除非用户明确选择上传 |
| **可以关闭所有通知** | 尊重用户的"断联权" |
| **不泄露练习频率** | 不向好友暴露"TA 多久没练了" |

### 11.3 每日练习上限

```
原则：宁可少练也不要伤嗓

系统行为：
  if 本日练习时长 > 30 min → 提醒"休息一下"
  if 本日练习时长 > 60 min → 强制休息 15 分钟（不计分、不计 XP）
  if 连续 7 天每天 > 45 min → "你的声音好努力！但记得给TA放个假"
```
