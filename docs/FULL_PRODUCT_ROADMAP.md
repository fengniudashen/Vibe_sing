# Vibesing（高音觉醒）— 完整产品路线图 V0.1 → V3.0

> **本文档定义从 MVP 到最终形态的全部技术与产品演进路径。**  
> **时间线为预估值，实际进度取决于数据采集速度和用户反馈。**  
> **日期**：2026-03-30

---

## 产品愿景（终态）

> **每个人都有高音，只是没有觉醒。**

Vibesing 的最终形态是一个**离线端侧 AI 声乐教练**，能够：

1. 实时诊断 7 个维度的发声状态（机能、混声、病灶、共鸣、修饰、代偿、训练态）
2. 像排雷一样逐个消除阻碍高音的生理代偿动作
3. 通过训练引导模式帮用户建立混声通道
4. 输出可量化的进步轨迹（周报/月报）

## V0.2 — 混声检测

**目标**：在挤卡检测之上，增加胸声/混声/头声的 3 级分类 Slider。

| 项目 | 详情 |
|------|------|
| **核心功能** | 实时 Slider 显示当前声区（胸声主导 / 混声 / 头声主导） |
| **技术实现** | 5-class 有序分类 Softmax → 期望值映射为连续 Slider |
| **深度学习** | 首次引入 DL：单流 Causal CNN，2 个输出头（挤卡 + 混声） |
| **输入特征** | Mel(80) + HNR + Energy (与 V0.1 相同) |
| **新增 UI** | 胸/混/头 Slider + 颜色渐变（蓝→绿→紫） |
| **数据需求** | V0.1 TestFlight 回收数据 + VocalSet + Mixup 合成（~2000 条） |
| **训练策略** | Rule-based 伪标签启动 → 人工标注修正 → Label Smoothing ε=0.1 |
| **成功指标** | 混声 3-class F1 >0.7；模型 vs Rule-based 挤卡准确率增益 >5% |
| **时间** | +2 周（累计 4 周） |

### 关键技术
```python
# 5-class 混声有序分类
# ["full_chest", "chest_mix", "balanced", "head_mix", "full_head"]
bin_centers = [0.1, 0.3, 0.5, 0.7, 0.9]
probs = softmax(logits)      # [5]
mix_ratio = sum(probs * bin_centers)  # 连续值 → UI Slider
```

### 模型架构升级
```
V0.1: Rule-based (no model)
  ↓
V0.2: Single-stream Causal CNN
      Input: [82 × 32]
      Head A: squeeze (sigmoid)
      Head B: mix_class (5-softmax)
      Params: ~300K
```

## V1.0 — 多维诊断引擎（正式版）

**目标**：完整实现 Taxonomy V1.0 的前 5 个维度，从"单一病灶检测"升级为"全面发声诊断"。

| 项目 | 详情 |
|------|------|
| **覆盖维度** | D1(机能) + D2(混声) + D3(全部病灶) + D4(共鸣) + D5(时序事件) |
| **技术升级** | **双流 Late Fusion 架构**（Mel stream + Physics stream） |
| **新增特征** | CPP（后端稳定版，中值滤波 + voicing mask） + 频谱斜率 |
| **新增 UI** | 诊断报告页面 + 雷达图（7 维得分）+ 病灶时间线回放 |
| **平滑系统** | 完整版 VocalHealthScore（EMA + 惩罚脉冲 + 持续奖励 + 状态机） |
| **底层能力** | CoreML Neural Engine 优化（Float16, fixed shape, fused BN） |
| **数据需求** | ~10000 条多标签标注数据 |
| **成功指标** | 全维度 macro-F1 >0.65；诊断报告被声乐老师认可 |
| **时间** | +2 月（累计 ~3 个月） |

### 模型架构
```
Stream A (频谱流):                  Stream B (物理流):
  80-Mel → Causal CNN (4 blocks)     [CPP, HNR, H1-H2, Energy, SpectralSlope]
  → 256-dim                          → MLP(5→64→128) → 128-dim
              ↘                    ↙
               Concat → 384-dim
                    ↓
        Multi-Task Heads (×7+)
        ├── D1: register (4-softmax)
        ├── D2: mix_class (5-softmax)
        ├── D3: squeeze (sigmoid)
        ├── D3: breathy (sigmoid)
        ├── D3: yelling (sigmoid)
        ├── D3: high_larynx (sigmoid)
        ├── D4: resonance (4-softmax)
        └── D5: break_event (sigmoid)

Params: ~800K
Latency: <1ms on A15+ Neural Engine
```

### 训练策略升级
- Multi-Annotator Distribution Loss（混声比例）
- 物理一致性正则化（F0 vs 混声比例的物理约束）
- Co-Teaching（抗标注噪音）
- Temperature Scaling（概率校准）

### 破音/颤音 → 规则引擎（不进模型）
```swift
// 破音：F0 帧间差 > 阈值
if abs(f0[t] - f0[t-1]) > 50 Hz && both voiced → break event

// 颤音：F0 调制检测
if f0_modulation_freq ∈ [4.5, 6.5] Hz && depth ∈ [0.5, 2] semitones → vibrato
```

## V2.0 — 训练引导模式

**目标**：从"诊断工具"进化为"训练教练"——不仅告诉你哪里有问题，还引导你做练习来修复。

| 项目 | 详情 |
|------|------|
| **核心功能** | 交互式练声模式：App 给出练习指令 → 用户执行 → 实时评估 → 即时反馈 |
| **新增维度** | D6(隐蔽代偿) + D7(训练态评估) |
| **训练课程** | 预设 50-100 个练声动作（唇颤音、哼鸣、Nay 混声、咽壁集中度等） |
| **智能选课** | 基于用户诊断结果自动推荐最需要的练习 |
| **进步追踪** | 周报/月报：各维度得分趋势图 + 里程碑成就 |
| **社交** | 练声日记分享（可选） |
| **时间** | +3 月（累计 ~6 个月） |

### 训练态检测
```
D7 新增检测：
- 唇颤音(Lip Trill): 频谱 ~20-30Hz 调制检测
- 哼鸣(Humming): 鼻腔共振能量 + 极低开口度
- "Nay" 咽壁集中: 3kHz Singer's Formant 强度评估
- 滑音练习(Scale): F0 轨迹 vs 目标音高曲线对齐度
```

### 隐蔽代偿检测
```
D6 新增检测：
- 舌根后压: F2 异常下降（相对于该元音的标准 F2）
  → 需要引入元音识别模块（轻量 ASR 子模型）
- 下巴僵硬: 辅音→元音过渡时间 > 阈值
  → 需要音素边界检测
```

### 练声交互设计
```
App 提示："请做唇颤音（打嘟噜），从 C4 滑到 G4"
    ↓
用户执行
    ↓
实时反馈：
  - ✅ 唇颤音稳定（frequency modulation 稳定）
  - ⚠️ 中间断了（能量骤降检测）
  - ✅ 音准良好（F0 vs 目标 < 50 cents）
    ↓
评分 + "再试一次" / "进入下一个练习"
```

## V3.0 — AI 个性化教练（终态）

**目标**：每个用户拥有一个"懂自己嗓子"的 AI 教练，能适应个体差异并持续优化训练方案。

| 项目 | 详情 |
|------|------|
| **核心功能** | 个性化训练方案 + 自适应难度 + 语音交互 |
| **AI 能力** | 端侧小模型（声学诊断）+ 云端大模型（训练方案生成、自然语言反馈） |
| **个性化** | 基于用户历史数据的 few-shot 适应：学习每个用户的换声点、音域、常见代偿模式 |
| **语音教练** | 语音交互：用户唱完一句 → AI 用语音说"你刚才 E4 的时候稍微挤了一下，试着把喉头放低再来" |
| **跨平台** | iOS + Android + Web（核心 DSP/AI 模型共享 C++ 底层） |
| **商业化** | 免费版（基础诊断）+ 订阅制（AI 教练 + 训练课程 + 进步报告） |
| **时间** | +6 月（累计 ~12 个月） |

## 关键风险与应对

| 风险 | 影响 | 概率 | 应对 |
|------|------|------|------|
| 用户不在乎声学诊断 | 产品方向错误 | 中 | V0.1 TestFlight 快速验证 PMF |
| HNR 在真实场景不稳定 | 误报率高 → 用户流失 | 中 | 多环境测试 + 动态阈值适应 |
| DL 模型比 Rule-based 没有显著提升 | DL 投入浪费 | 低 | 如果增益 <5% 就保留 Rule-based |
| 苹果 Neural Engine API 变更 | CoreML 模型需重新适配 | 低 | 抽象推理接口 + 保留 CPU fallback |
| 竞品抄袭核心功能 | 先发优势缩小 | 中 | 数据飞轮（用户越多→数据越多→模型越准→用户越多） |
| 标注数据不足导致多维模型效果差 | V1.0 延期 | 高 | TestFlight 数据回收 + 半监督 + 数据增强 |
