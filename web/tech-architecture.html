🎵 F0 提取 (Pitch Detection)

自相关法 · 50-800Hz · 每帧 16ms
window = audio[ring_buffer] # 1024 samples acf = autocorrelate(window * hann) # 搜索范围: 50Hz → 800Hz # → lag_min = SR/800 = 20 # → lag_max = SR/50 = 320 peak_lag = argmax(acf[20:320]) f0 = SR / peak_lag # 可信度: acf_peak / acf[0] confidence = acf[peak_lag] / acf[0] if confidence < 0.5: f0 = 0 # unvoiced
复杂度O(N²) ≈ O(1M)
端侧延迟< 0.5ms (vDSP)
精度±1Hz (稳态元音)

📊 HNR 提取 (Harmonics-to-Noise Ratio)

自相关法分解 · 声质核心指标
# 在 F0 周期处的自相关值 harmonic_energy = acf[f0_lag] total_energy = acf[0] noise_energy = total_energy - harmonic_energy hnr_db = 10 * log10( harmonic_energy / max(noise_energy, 1e-10) ) # 正常: > 18dB # 挤卡: < 12dB # 严重: < 6dB
与 F0 的关系依赖 F0 → 必须先算 F0
V0.1 用途挤卡检测的核心输入
局限噪声环境下偏低 → 动态阈值

🧠 V0.1 规则引擎 (Squeeze Detection)

sigmoid + EMA + 状态机 · 零数据即可运行
# Gate if energy < -40 or f0 < 50: prob = 0; return # silence / unvoiced # Core: HNR → probability prob = sigmoid(0.5 * (THRESHOLD - hnr)) # THRESHOLD = dynamic (user_baseline - 6) # Pitch boost if f0 > 220: boost = 1 + 0.3 * min((f0-220)/300, 1) prob = min(prob * boost, 1.0) # EMA smooth ema = α*prob + (1-α)*ema_prev # α=0.4 # State machine if ema > 0.55 for 6+ frames → RED if ema < 0.30 for 10+ frames → GREEN
算法类别Rule-based
参数数量0 (纯规则)
精度预期65-75%
关键改进 (修订版)动态阈值 + 环境噪声防护

🧠 V0.2 CNN 模型 (DL Squeeze Detection)

Causal DWSep CNN · 280K params · CoreML Float16
# Input: [batch=1, channels=82, frames=32] # 82 = 80 Mel bands + HNR + Energy # 32 = 32 frames × 16ms = 512ms context Layer 1: DWSep(82→128, k=3, d=1) + BN + ReLU Layer 2: DWSep(128→128, k=3, d=2) + BN + ReLU Layer 3: DWSep(128→128, k=3, d=4) + BN + ReLU Layer 4: DWSep(128→64, k=3, d=8) + BN + ReLU Head: Conv1D(64→1, k=1) → sigmoid # 感受野: 31 frames = 496ms ← 覆盖完整发声周期 # Causal padding: 只用过去帧，不看未来
算法类别Deep Learning
模型大小560KB (Float16)
推理延迟~1.5ms (A14 Neural Engine)
训练数据需求500+ labeled clips

📈 后处理: EMA + 状态机

信号层平滑 + 语义层去抖 · 双层防护
┌─────── L1: EMA (信号层) ───────┐ │ │ │ raw: 0.8 0.3 0.9 0.2 0.7 ... │ │ ema: 0.8 0.6 0.7 0.5 0.6 ... │ │ α=0.4 → ~2.5 帧等效平滑 │ │ │ └─────────────────────────────────┘ ↓ ┌─────── L2: 状态机 (语义层) ─────┐ │ │ │ State: GREEN ──[6帧>0.55]──→ RED │ State: RED ──[10帧<0.30]──→ GREEN │ │ │ 效果: 红灯不闪烁，切换有"惯性" │ │ 6帧 = 96ms 触发 (用户感知自然) │ │ 10帧 = 160ms 恢复 (不过早放行) │ └─────────────────────────────────┘
设计原则宁可漏报，不可闪烁
滞后控制ON: 0.55 / OFF: 0.30 (磁滞)

🔇 环境噪声估计

底噪追踪 · 自动降级 · 避免误报
# 追踪最近 5 秒内能量最低的 10% 帧 noise_floor = percentile(energy_buffer, 10) # 分级策略: if noise < -45dB → 🟢 全功能 if noise < -35dB → 🟡 阈值放宽 2dB if noise < -25dB → 🟠 只输出 Energy 指标 if noise > -25dB → 🔴 "环境太吵" # 动态阈值: threshold = max(user_baseline_hnr - 6, 8.0) # 在嘈杂环境下自动提高阈值
目的防止"没挤卡但一直红灯"
原则宁可不说，不可说错
